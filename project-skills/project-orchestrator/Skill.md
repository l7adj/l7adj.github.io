# Project Orchestrator Skill

## الهدف

هذه المهارة هي المهارة الحاكمة للمشروع. تستخدم قبل أي Skill أخرى لضمان أن العمل النهائي متوافق مع:

- قالب الموقع
- قواعد SEO
- صفحات الثقة
- GitHub Pages
- جودة البيانات
- الروابط الداخلية
- اللغة والأسلوب
- بنية الملفات

## متى تستخدم؟

استخدمها دائمًا عند أي طلب متعلق بـ:

- إنشاء مراجعة شركة
- تحديث مراجعة شركة
- إنشاء مقارنة
- تعديل صفحة موجودة
- إضافة شركة جديدة
- تحسين SEO
- تجهيز صفحة للنشر
- تحويل مقال إلى HTML
- تدقيق بيانات

## ترتيب تشغيل المهارات

### عند إنشاء مراجعة شركة

شغّل بالترتيب:

1. `data-quality`
2. `company-review`
3. `seo-publisher`
4. `github-pages`

### عند إنشاء مقارنة

شغّل بالترتيب:

1. `data-quality`
2. `company-comparison`
3. `seo-publisher`
4. `github-pages`

### عند تعديل صفحة موجودة

شغّل بالترتيب:

1. `data-quality` إذا كان التعديل يمس الأرقام أو القواعد
2. `seo-publisher` إذا كان التعديل يمس العنوان أو الوصف أو الروابط
3. `github-pages` قبل النشر

## قواعد التوافق الإجباري

أي صفحة نهائية يجب أن تحقق التالي:

- تستخدم نفس الهيدر والفوتر الموجودين في الموقع.
- تحتوي رابط CSS الصحيح:
  `/assets/css/style.css`
- تحتوي رابط JS الصحيح:
  `/assets/js/app.js`
- تستخدم canonical من الدومين الرسمي:
  `https://l7adj.github.io/`
- لا تحتوي أي رابط قديم إلى:
  - `prophub.com`
  - `singular-unicorn-ed0891.netlify.app`
  - `sit-trader-platform.netlify.app`
- تحتوي تنبيه مخاطر عند الحديث عن التداول.
- تحتوي روابط صفحات الثقة في الفوتر.
- لا تحتوي أرقامًا غير مؤكدة.
- لا تستخدم عبارات تسويقية مبالغ فيها.

## بنية الملفات

استخدم هذه المسارات فقط:

- مراجعات الشركات:
  `reviews/company-name.html`

- المقارنات:
  `comparisons/company-a-vs-company-b.html`

- التصنيفات:
  `categories/category-name.html`

- الأدوات:
  `tools/index.html`

- صفحات الثقة:
  - `about.html`
  - `methodology.html`
  - `privacy-policy.html`
  - `disclaimer.html`
  - `contact.html`

## نظام الحكم على الجودة

قبل نشر أي صفحة، أعطها تقييمًا من 10 حسب التالي:

- دقة البيانات: 0-2
- وضوح المقال: 0-2
- توافق القالب: 0-2
- SEO: 0-2
- الثقة والتحذيرات والروابط: 0-2

لا تنشر صفحة أقل من 8/10 إلا إذا قال المستخدم صراحة: انشرها رغم النقص.

## مخرجات أي مهمة

في نهاية أي تنفيذ، أخرج للمستخدم:

1. ما الذي تم إنجازه.
2. الملفات التي تم تعديلها أو إضافتها.
3. هل تحتاج الصفحة بيانات ناقصة؟
4. درجة الثقة.
5. الخطوة التالية فقط.

## قاعدة منع الفوضى

لا تنشئ نسخًا متعددة من نفس الملف.
لا تنشئ ZIP إلا إذا طلب المستخدم ذلك صراحة.
لا تغيّر بنية المشروع إلا إذا كان ذلك ضروريًا.
لا تخلط بين Netlify وGitHub Pages. المشروع الرسمي الآن يعمل على:

`https://l7adj.github.io/`
