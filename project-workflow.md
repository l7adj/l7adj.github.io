# Project Workflow - محطة التمويل

هذا هو نظام العمل الرسمي للمشروع. الهدف هو إنجاز المحتوى بسرعة ودقة وبدون أي أدوات مدفوعة.

## القاعدة العامة

- الاستضافة الرسمية: GitHub Pages فقط.
- الرابط الرسمي: https://l7adj.github.io/
- لا نستخدم Netlify.
- لا نستخدم GitHub Actions.
- لا نستخدم Build commands.
- لا نستخدم npm أو أدوات مدفوعة.
- الموقع ثابت HTML/CSS/JS فقط.

## Workflow مراجعة شركة

1. تحديد الشركة.
2. جمع البيانات من المصادر الرسمية أو المقال الذي يرسله المستخدم.
3. فصل البيانات المؤكدة عن غير المؤكدة.
4. تطبيق قالب مراجعة الشركة.
5. إنشاء الجداول الثابتة.
6. كتابة الحكم النهائي.
7. إضافة FAQ.
8. ضبط SEO.
9. إنشاء أو تحديث ملف داخل `reviews/`.
10. فحص الروابط على GitHub Pages.

## Workflow مقارنة شركتين

1. تحديد الشركتين.
2. جمع بيانات كل شركة.
3. توحيد الأرقام في جدول واحد.
4. تحديد الفائز في كل بند.
5. كتابة الحكم النهائي.
6. ضبط SEO.
7. إنشاء أو تحديث ملف داخل `comparisons/`.

## Workflow SEO

1. H1 واضح.
2. title مختلف لكل صفحة.
3. meta description مختلف لكل صفحة.
4. canonical على `https://l7adj.github.io/`.
5. روابط داخلية.
6. روابط صفحات الثقة في الفوتر.
7. تنبيه مخاطر في صفحات التداول.
8. تحديث sitemap عند إضافة صفحات مهمة.

## Workflow GitHub Pages

قبل أي تعديل تأكد من:

- لا توجد روابط إلى Netlify.
- لا توجد روابط إلى prophub.com.
- روابط CSS وJS تبدأ من الجذر `/assets/...` لأن المستودع اسمه `l7adj.github.io`.
- ملف `.nojekyll` موجود.
- `index.html` في الجذر.
- `robots.txt` يشير إلى `https://l7adj.github.io/sitemap.xml`.

## مخرجات كل مهمة

عند الانتهاء، يجب ذكر:

1. الملفات التي تم تعديلها.
2. النتيجة العملية.
3. هل توجد بيانات ناقصة؟
4. الخطوة التالية فقط.
